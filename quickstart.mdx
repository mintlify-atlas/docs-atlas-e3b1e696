Get LORE Timeline up and running and create your first interactive biography timeline in under 5 minutes.

## What You'll Build

By the end of this quickstart, you'll have:
- A fully functional LORE Timeline instance running locally
- Your first person timeline with AI-generated life stages
- An understanding of how to navigate and use the platform

## Prerequisites

Before you begin, make sure you have:
- **Node.js 18.x or higher** installed
- **OpenAI API key** (get one at [platform.openai.com](https://platform.openai.com/api-keys))
- **Exa API key** (optional, get one at [exa.ai](https://exa.ai/))

## Step 1: Clone and install

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/patelpurav05/timeline.git
    cd timeline
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash pnpm
      pnpm install
      ```

      ```bash yarn
      yarn install
      ```
    </CodeGroup>
  </Step>

  <Step title="Initialize Convex">
    ```bash
    npx convex dev
    ```
    
    This will:
    - Create a new Convex project
    - Set up the database schema
    - Open the Convex dashboard
    - Generate environment files
  </Step>
</Steps>

## Step 2: Configure environment

<Steps>
  <Step title="Set up Convex environment variables">
    1. Go to the [Convex dashboard](https://dashboard.convex.dev)
    2. Navigate to **Settings** → **Environment Variables**
    3. Add the following variables:
    
    ```bash
    OPENAI_API_KEY=sk-proj-your-key-here
    EXA_API_KEY=your-exa-key-here  # Optional
    ```
  </Step>

  <Step title="Restart Convex">
    ```bash
    # Stop the dev server (Ctrl+C)
    npm run dev
    ```
  </Step>
</Steps>

<Note>
Environment variables for Convex backend functions must be set in the Convex dashboard, not in `.env.local`
</Note>

## Step 3: Start the app

```bash
npm run dev
```

This starts both:
- **Next.js frontend** on [http://localhost:3000](http://localhost:3000)
- **Convex backend** watching for changes

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Step 4: Create your first timeline

<Steps>
  <Step title="Enter a person's name">
    On the homepage, enter a name in the search box. Try someone well-known like:
    - "Sam Altman"
    - "Steve Jobs"
    - "Elon Musk"
  </Step>

  <Step title="Watch the pipeline">
    LORE Timeline will automatically:
    1. **Discover** sources via Exa AI (articles, interviews, videos)
    2. **Extract** content and YouTube transcripts
    3. **Stage** life eras with AI-generated summaries
    4. **Embed** text chunks for semantic search
    5. **Publish** timeline cards with images
    
    This takes **3-8 minutes** depending on the person.
  </Step>

  <Step title="Explore the timeline">
    Once ready, you'll see:
    - **Scrollable timeline** with life stages
    - **Stage summaries** with turning points and worldview
    - **Timeline cards** with moments, quotes, and media
    - **Sources** with citations and previews
  </Step>

  <Step title="Chat with the person">
    Click the **Chat** button to start a conversation:
    
    - Ask about specific eras: "What were you building in your early 20s?"
    - Explore turning points: "What motivated you to start Y Combinator?"
    - Understand worldview: "How did your thinking evolve over time?"
    
    All responses are grounded in real evidence with inline citations.
  </Step>
</Steps>

## Common questions

<AccordionGroup>
  <Accordion title="How long does it take to generate a timeline?">
    **3-8 minutes** depending on:
    - How well-known the person is (more sources = longer processing)
    - Number of YouTube videos with transcripts
    - Exa API response time
    
    You can monitor progress in real-time via the progress tracker.
  </Accordion>

  <Accordion title="Can I generate timelines for anyone?">
    Yes! LORE Timeline works best for:
    - **Public figures** with articles, interviews, or podcasts
    - **Historical figures** with biographical content
    - **Industry leaders** with talks and media appearances
    
    Less effective for:
    - Private individuals with no public information
    - Fictional characters
    - People with very common names (hard to disambiguate)
  </Accordion>

  <Accordion title="What if the pipeline fails?">
    The pipeline is resumable. If a phase fails:
    1. Check Convex logs for the error message
    2. Fix the issue (usually missing API keys or rate limits)
    3. Use `reingestPerson` to resume from that phase
    
    See [How It Works](/how-it-works) for error handling details.
  </Accordion>

  <Accordion title="Can I customize the timeline?">
    Yes! You can customize:
    - **UI theme** (colors, fonts) via CSS variables
    - **Pipeline behavior** (number of sources, chunk sizes) in `convex/pipeline.ts`
    - **LLM models** via environment variables
    
    See [Customization](/guides/customization) for details.
  </Accordion>

  <Accordion title="How accurate are the timelines?">
    Timelines are **only as accurate as the sources**. LORE Timeline:
    - Cites all sources inline
    - Never fabricates information
    - Uses AI to organize and summarize, not invent
    
    Always verify important facts against the cited sources.
  </Accordion>
</AccordionGroup>
