## Overview

LORE Timeline is built on [Convex](https://convex.dev), a real-time backend-as-a-service platform that provides:

- **Type-safe database** with automatic schema validation
- **Real-time queries** that automatically update UI when data changes
- **Serverless functions** (queries, mutations, actions)
- **Vector storage** for semantic search and RAG
- **Scheduled jobs** for long-running pipeline operations

The backend handles the entire biography ingestion pipeline, from source discovery to timeline generation, plus real-time chat interactions with the AI persona.

## Core Concepts

Convex organizes server-side code into three function types:

<Tabs>
  <Tab title="Queries">
    **Read-only** functions that fetch data from the database.

    - Return data to the client
    - Cannot modify database state
    - Automatically reactive (UI updates when data changes)
    - Can be called from the frontend

    Example from `convex/persons.ts:176`:

    ```typescript
    export const getPerson = query({
      args: { personId: v.id("persons") },
      handler: async (ctx, args) => {
        const person = await ctx.db.get(args.personId);
        if (!person) return null;

        const jobs = await ctx.db
          .query("jobs")
          .withIndex("by_person", (q) => q.eq("personId", args.personId))
          .collect();

        return { person, jobs };
      },
    });
    ```
  </Tab>

  <Tab title="Mutations">
    **Write operations** that modify database state.

    - Insert, update, or delete database records
    - Transactional and atomic
    - Can schedule actions
    - Can be called from the frontend

    Example from `convex/persons.ts:14`:

    ```typescript
    export const createPerson = mutation({
      args: {
        name: v.string(),
        seedUrls: v.optional(v.array(v.string())),
      },
      handler: async (ctx, args) => {
        const personId = await ctx.runMutation(api.pipeline.bootstrapPerson, {
          name: args.name,
          seedUrls: args.seedUrls,
        });
        await ctx.runMutation(api.pipeline.startIngestion, { personId });
        return { personId, existing: false };
      },
    });
    ```
  </Tab>

  <Tab title="Actions">
    **Long-running operations** that can call external APIs.

    - Can make HTTP requests (OpenAI, Exa, YouTube)
    - Can call queries and mutations
    - Cannot directly access the database
    - Run in Node.js environment

    Example from `convex/llm.ts:47`:

    ```typescript
    export const embedText = action({
      args: { text: v.string() },
      handler: async (_ctx, args) => {
        const response = await fetch("https://api.openai.com/v1/embeddings", {
          method: "POST",
          headers: {
            Authorization: `Bearer ${OPENAI_API_KEY}`,
            "Content-Type": "application/json",
          },
          body: JSON.stringify({
            model: OPENAI_EMBED_MODEL,
            input: args.text,
          }),
        });
        const data = await response.json();
        return data.data[0]?.embedding ?? [];
      },
    });
    ```
  </Tab>
</Tabs>

## Schema Definition

The database schema is defined in `convex/schema.ts` using Convex's type-safe schema builder:

```typescript
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  persons: defineTable({
    name: v.string(),
    slug: v.string(),
    status: v.union(
      v.literal("pending"),
      v.literal("processing"),
      v.literal("ready"),
      v.literal("failed"),
    ),
    birthDate: v.optional(v.string()),
    createdAt: v.number(),
    updatedAt: v.number(),
  })
    .index("by_slug", ["slug"])
    .index("by_created", ["createdAt"]),
  // ... more tables
});
```

Indexes enable efficient queries. For example, `by_slug` allows fast lookups by URL slug, and `by_created` enables chronological sorting.

See [Database Schema](/development/database-schema) for complete table definitions.

## Person Management

The `convex/persons.ts` file handles person-related operations:

### Creating a Person

```typescript
// Client-side call
const { personId } = await convex.mutation(api.persons.createPerson, {
  name: "Steve Jobs",
  seedUrls: ["https://en.wikipedia.org/wiki/Steve_Jobs"],
});
```

This mutation:
1. Checks for existing person with same name
2. Calls `bootstrapPerson` to create database record
3. Schedules the ingestion pipeline via `startIngestion`
4. Returns the new `personId`

### Querying Timeline Data

```typescript
export const getTimeline = query({
  args: { personId: v.id("persons") },
  handler: async (ctx, args) => {
    const stages = await ctx.db
      .query("stages")
      .withIndex("by_person_order", (q) => q.eq("personId", args.personId))
      .order("asc")
      .collect();

    const timeline = [];
    for (const stage of stages) {
      const cards = await ctx.db
        .query("timelineCards")
        .withIndex("by_stage", (q) => q.eq("stageId", stage._id))
        .collect();
      timeline.push({ stage, cards });
    }
    return timeline;
  },
});
```

This query demonstrates:
- **Indexed queries** for efficient filtering (`by_person_order`)
- **Ordered results** (`order("asc")`)
- **Relational data** (joining stages with their cards)

## Internal Mutations

Internal mutations can only be called from backend code (actions, other mutations). They're marked with `internalMutation` and used for pipeline operations:

```typescript
export const markPersonStatus = internalMutation({
  args: {
    personId: v.id("persons"),
    status: v.union(
      v.literal("pending"),
      v.literal("processing"),
      v.literal("ready"),
      v.literal("failed")
    ),
  },
  handler: async (ctx, args) => {
    await ctx.db.patch(args.personId, {
      status: args.status,
      updatedAt: Date.now(),
    });
  },
});
```

Internal functions provide:
- **Encapsulation**: Pipeline logic stays server-side
- **Security**: Cannot be called directly from client
- **Modularity**: Reusable building blocks for complex operations

## Scheduled Jobs

Convex's scheduler enables long-running pipeline operations:

```typescript
export const startIngestion = mutation({
  args: { personId: v.id("persons") },
  handler: async (ctx, args) => {
    await ctx.scheduler.runAfter(0, api.pipeline.runIngestion, {
      personId: args.personId,
      phase: "discover",
    });
  },
});
```

Key features:
- **Asynchronous execution**: Returns immediately, work happens in background
- **Phase chaining**: Each phase schedules the next upon completion
- **Resumability**: Phases can be restarted if they fail or timeout

From `convex/pipeline.ts:828`:


## Vector Search

Convex stores embeddings as arrays of floats, enabling semantic search:

```typescript
export const scoreAndSelectChunks = action({
  args: {
    personId: v.id("persons"),
    stageId: v.id("stages"),
    query: v.string(),
    topK: v.optional(v.number()),
  },
  handler: async (ctx, args) => {
    const qEmbedding = await ctx.runAction(api.llm.embedText, { text: args.query });
    
    const stageChunks = await ctx.runQuery(internal.pipeline.listChunksByStage, {
      stageId: args.stageId,
    });
    
    const scored = stageChunks
      .map(chunk => ({
        chunk,
        score: cosineSimilarity(qEmbedding, chunk.embedding)
      }))
      .sort((a, b) => b.score - a.score)
      .slice(0, args.topK ?? 8);
    
    return { chunks: scored };
  },
});
```

The `cosineSimilarity` function (from `convex/pipeline.ts:140`) computes relevance:

```typescript
function cosineSimilarity(a: number[], b: number[]): number {
  let dot = 0, magA = 0, magB = 0;
  for (let i = 0; i < a.length; i++) {
    dot += a[i] * b[i];
    magA += a[i] * a[i];
    magB += b[i] * b[i];
  }
  return dot / (Math.sqrt(magA) * Math.sqrt(magB));
}
```

## Error Handling

The pipeline includes comprehensive error handling with status tracking:

```typescript
try {
  // ... pipeline phases
} catch (error) {
  await ctx.runMutation(internal.pipeline.markPersonStatus, {
    personId: args.personId,
    status: "failed",
  });
  await ctx.runMutation(internal.pipeline.markLatestRunningJobFailed, {
    personId: args.personId,
    message: error instanceof Error ? error.message : "Unknown ingestion failure",
  });
  throw error;
}
```

LLM actions use retry logic for transient failures:

```typescript
async function fetchWithRetry(url: string, init: RequestInit, maxRetries = 3) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      const response = await fetch(url, init);
      if ([429, 502, 503].includes(response.status) && attempt < maxRetries) {
        const delay = Math.min(1000 * Math.pow(2, attempt), 8000);
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }
      return response;
    } catch (err) {
      if (attempt < maxRetries) {
        const delay = Math.min(1000 * Math.pow(2, attempt), 8000);
        await new Promise(resolve => setTimeout(resolve, delay));
      }
    }
  }
  throw new Error("Request failed after retries");
}
```
