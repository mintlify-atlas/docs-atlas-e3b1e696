## Overview

LORE Timeline uses **Tailwind CSS v4** with a custom **comic book aesthetic**. The design system emphasizes bold borders, offset shadows, and a warm paper-like color palette.

## Tailwind Configuration

LORE uses **Tailwind v4** with the new CSS-first configuration approach.

### PostCSS Setup

```js postcss.config.mjs
const config = {
  plugins: ["@tailwindcss/postcss"],
};

export default config;
```

### Global CSS Import

```css app/globals.css
@import "tailwindcss";

/* CSS variables and custom styles below */
```

Tailwind v4 is imported directly in CSS using the `@import` directive instead of traditional `@tailwind` directives.

## CSS Variables

The entire color palette is defined using CSS custom properties in `app/globals.css`.

<CodeGroup>
```css app/globals.css
:root {
  --paper: #f4efe7;       /* Warm cream background */
  --ink: #171310;          /* Near-black text */
  --panel: #fffaf2;        /* Off-white panel backgrounds */
  --accent: #c34628;       /* Rust red for CTAs and highlights */
  --frame: #2c2420;        /* Dark brown borders */
  --muted: #a09888;        /* Muted text for labels */
  --success: #2d6a4f;      /* Green for success states */
  --warn: #b5851b;         /* Amber for warnings */
}
```
</CodeGroup>

### Color Usage

| Variable | Usage | Examples |
|----------|-------|----------|
| `--paper` | Page background | `<body>` background |
| `--ink` | Primary text | Headings, body text |
| `--panel` | Card backgrounds | Timeline cards, modals |
| `--accent` | CTAs, links | Buttons, active states |
| `--frame` | Borders | Comic panel borders, dividers |
| `--muted` | Secondary text | Labels, timestamps |
| `--success` | Success indicators | Completed phases |
| `--warn` | Warning indicators | Low confidence, processing |

### Skeleton Loader

```css app/globals.css
.skeleton {
  background: linear-gradient(
    90deg,
    var(--panel) 25%,
    var(--paper) 50%,
    var(--panel) 75%
  );
  background-size: 200% 100%;
  animation: skeleton-pulse 1.5s ease-in-out infinite;

@keyframes skeleton-pulse {
  0% {
    background-position: 200% 0;
  }
  100% {
    background-position: -200% 0;
  }
}

@media (prefers-reduced-motion: reduce) {
  .skeleton {
    animation: none;
  }
}
```

**Usage:**

```tsx

    </div>
  );
}
```

## Utility Patterns

### Responsive Design

LORE uses Tailwind's responsive prefixes consistently:

```tsx
{/* Mobile-first approach */}
<div className="px-5 py-4 sm:px-8 lg:px-12">
  {/* 5 → 8 → 12 as viewport grows */}
</div>

{/* Grid breakpoints */}
<div className="grid gap-4 sm:grid-cols-2 lg:grid-cols-3">
  {/* 1 → 2 → 3 columns */}
</div>

{/* Hidden on mobile */}
<nav className="hidden lg:block">
  {/* Desktop-only sidebar */}
</nav>
```

### Conditional Styling with `cn()`

The `cn()` utility combines `clsx` and `tailwind-merge` for conditional classes.

```tsx
import { cn } from "@/lib/cn";

<div className={cn(
  "rounded-xl border-2 px-4 py-3",
  status === "ready" && "border-[var(--success)] text-[var(--success)]",
  status === "processing" && "border-[var(--warn)] text-[var(--warn)]",
  status === "failed" && "border-red-500 text-red-500"
)} />
```

### Custom Color Usage

Access CSS variables using bracket notation:

```tsx
{/* Background */}
<div className="bg-[var(--panel)]" />

{/* Text color */}
<p className="text-[var(--muted)]" />

{/* Border */}
<div className="border-2 border-[var(--frame)]" />

{/* Opacity */}
<div className="text-[var(--ink)]/80" />
```

## Design Tokens

### Spacing Scale

LORE uses Tailwind's default spacing scale with these common values:

- **xs**: `0.125rem` (2px)
- **sm**: `0.25rem` (4px)
- **md**: `0.5rem` (8px)
- **lg**: `0.75rem` (12px)
- **xl**: `1rem` (16px)
- **2xl**: `1.5rem` (24px)

### Border Widths

```css
border      /* 1px */
border-2    /* 2px - Most common for comic panels */
border-3    /* 3px - Comic panel cards */
border-4    /* 4px - Hero panels */
```

### Border Radius

```css
rounded-lg     /* 0.5rem - Standard inputs */
rounded-xl     /* 0.75rem - Buttons, cards */
rounded-2xl    /* 1rem - Speech bubbles */
rounded-3xl    /* 1.5rem - Hero panels */
rounded-full   /* 9999px - Badges, dots */
```

### Shadows

Custom shadows are defined inline for comic effect:

```tsx
{/* Offset shadow (comic panel) */}
<div className="shadow-[5px_5px_0_0_var(--frame)]" />

{/* Large offset shadow (hero panel) */}
<div className="shadow-[8px_8px_0_0_var(--frame)]" />

{/* Subtle hover shadow */}
<div className="hover:shadow-[3px_3px_0_0_var(--frame)]" />
```

## Accessibility

### Focus States

All interactive elements have visible focus states:

```tsx
{/* Input focus */}
<input className="outline-none focus:border-[var(--accent)]" />

{/* Button focus */}
<button className="focus:ring-2 focus:ring-[var(--accent)] focus:ring-offset-2" />
```

### Reduced Motion

Animations respect user preferences:

```css
@media (prefers-reduced-motion: reduce) {
  .skeleton {
    animation: none;
  }
}
```

Framer Motion automatically respects `prefers-reduced-motion`.

### Semantic HTML

- Use `<nav>` for navigation
- Use `<main>` for page content
- Use `<section>` for timeline chapters
- Use `<article>` for timeline cards
- Include `aria-label` on interactive elements

## Dark Mode

LORE currently uses a **fixed light theme** with warm, paper-like tones. Dark mode is not implemented but could be added using CSS variables:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --paper: #1a1613;
    --ink: #f4efe7;
    --panel: #2c2420;
    /* ... */
  }
}
```

## Best Practices

1. **Use CSS variables** for all colors via bracket notation: `bg-[var(--panel)]`
2. **Prefer Tailwind utilities** over custom CSS when possible
3. **Use `cn()` helper** for conditional classes to avoid conflicts
4. **Follow mobile-first** approach with `sm:`, `lg:` prefixes
5. **Respect reduced motion** - test with OS-level setting enabled
6. **Maintain comic aesthetic** - always use offset shadows and bold borders
7. **Keep spacing consistent** - use Tailwind's scale (px-4, py-3, gap-2, etc.)
