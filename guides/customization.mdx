### Customizing Fonts

<Steps>
  <Step title="Choose Fonts">
    Browse [Google Fonts](https://fonts.google.com) and select:
    - A **display font** for headings (bold, impactful)
    - A **body font** for text (readable, multiple weights)
  </Step>
  
  <Step title="Update Imports">
    Replace the font imports in `layout.tsx`:
    
    ```typescript
    import { Montserrat, Inter } from "next/font/google";
    
    const display = Montserrat({
      subsets: ["latin"],
      weight: ["700", "900"],
      variable: "--font-display",
    });
    
    const body = Inter({
      subsets: ["latin"],
      weight: ["400", "500", "600", "700"],
      variable: "--font-body",
    });
    ```
  </Step>
  
  <Step title="Update CSS">
    Font variables are already wired up in `globals.css:15, 32, 38`:
    
    ```css
    body {
      font-family: var(--font-body), sans-serif;
    }
    
    h1, h2, h3 {
      font-family: var(--font-display), sans-serif;
    }
    ```
    
    No changes needed here unless you want to adjust font stacks.
  </Step>
</Steps>

## Pipeline Behavior Customization

The ingestion pipeline can be customized in `~/workspace/source/convex/pipeline.ts`.

### Discovery Search Strategy

**Location**: `~/workspace/source/convex/pipeline.ts:751-771`

You can modify the LLM prompt that generates search queries:

<CodeGroup>
```typescript pipeline.ts
const queryPlanJson = await generateJson(
  `You are a research agent. Given a person's name, generate a comprehensive set of search queries to discover biographical sources about them.

The person could be ANYONE — a musician, athlete, politician, scientist, actor, entrepreneur, activist, author, etc. Tailor queries to their likely domain.

Output a JSON object: { "queries": [...] }

Each query object has:
- "query": the search string (use quotes around the person's name for precision)
- "type": "web" or "video"
- "dateEnd": optional ISO date string to find older/historical content (e.g. "2010-01-01")

Generate 8-12 queries total. Include:
1. 2-3 general biography/profile web searches
2. 2-3 domain-specific web searches (e.g. discography for musicians, match highlights for athletes, policy speeches for politicians)
3. 2-3 mainstream video searches (interviews, talks, appearances relevant to their field)
4. 2-3 underground/rare/early video searches (before they were famous, small venues, niche podcasts, campus events, early career footage)
5. 1 search specifically for OLD content (set dateEnd to filter for historical material)

IMPORTANT: Use the person's full name in quotes in every query for precision.`,
  JSON.stringify({ personName: person.name }),
);
```
</CodeGroup>

**Customization ideas**:
- Increase query count: Change "8-12 queries" to "15-20 queries" for deeper discovery
- Add domain hints: Provide additional context (e.g., "This person is a tech founder")
- Adjust query types: Favor videos over articles, or vice versa

### Custom Convex Functions

To add backend logic:

<Steps>
  <Step title="Create Function File">
    ```bash
    touch ~/workspace/source/convex/analytics.ts
    ```
  </Step>
  
  <Step title="Define Function">
    ```typescript
    import { query } from "./_generated/server";
    import { v } from "convex/values";
    
    export const getPersonStats = query({
      args: { personId: v.id("persons") },
      handler: async (ctx, args) => {
        const person = await ctx.db.get(args.personId);
        const sources = await ctx.db
          .query("sources")
          .withIndex("by_person", (q) => q.eq("personId", args.personId))
          .collect();
        const stages = await ctx.db
          .query("stages")
          .withIndex("by_person", (q) => q.eq("personId", args.personId))
          .collect();
        
        return {
          name: person?.name,
          sourceCount: sources.length,
          stageCount: stages.length,
        };
      },
    });
    ```
  </Step>
  
  <Step title="Use in Frontend">
    ```typescript
    import { useQuery } from "convex/react";
    import { api } from "@/convex/_generated/api";
    
    const stats = useQuery(api.analytics.getPersonStats, { personId });
    ```
  </Step>
  
  <Step title="Deploy">
    ```bash
    npx convex dev  # Auto-deploys in dev mode
    ```
  </Step>
</Steps>
