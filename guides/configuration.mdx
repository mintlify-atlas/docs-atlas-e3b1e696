LORE Timeline is configured through environment variables and code-based configuration. This guide covers all available configuration options.

## Environment Variables

Environment variables are split between Convex (backend) and Next.js (frontend).

---

## Convex Backend Variables

These variables must be set in the **Convex Dashboard** (Settings → Environment Variables). They are used by serverless functions in `~/workspace/source/convex/`.

### Required Variables

<ParamField path="OPENAI_API_KEY" type="string" required>
  Your OpenAI API key for GPT and embedding models.
  
  **Where to get**: [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
  
  **Used in**: `~/workspace/source/convex/llm.ts:6`
  
  **Powers**:
  - Structured JSON generation for search plans, staging, and source vetting
  - Text embeddings for semantic search
  - Chat completions for era-based conversations
  
  **Example**: `sk-proj-abc123...`
</ParamField>

<ParamField path="EXA_API_KEY" type="string" required>
  Your Exa API key for web search and discovery.
  
  **Where to get**: [exa.ai](https://exa.ai) (sign up and get API key from dashboard)
  
  **Used in**: `~/workspace/source/convex/pipeline.ts:12`
  
  **Powers**:
  - Discovery phase: finding biographical sources (articles, videos, interviews)
  - Deep research: era-specific content discovery
  - Image search: finding person photos for each stage
  
  **Example**: `exa_abc123...`
  
  <Warning>
  Without `EXA_API_KEY`, the pipeline falls back to generic Wikipedia and YouTube search URLs (see `~/workspace/source/convex/pipeline.ts:326-368`).
  </Warning>
</ParamField>

### Optional Variables

<ParamField path="OPENAI_MODEL" type="string" default="gpt-5.2">
  The OpenAI model to use for chat completions and structured JSON generation.
  
  **Used in**: `~/workspace/source/convex/llm.ts:7`
  
  **Supported values**:
  - `gpt-5.2` (default, most capable)
  - `gpt-4-turbo` (faster, slightly less capable)
  - `gpt-4` (stable, well-tested)
  - `gpt-3.5-turbo` (fastest, lowest cost, reduced quality)
  
  **Example**: `gpt-4-turbo`
  
  <Note>
  The pipeline uses structured JSON output (`response_format: { type: "json_object" }`), so ensure your chosen model supports JSON mode (GPT-4+ models do).
  </Note>
</ParamField>

<ParamField path="OPENAI_EMBED_MODEL" type="string" default="text-embedding-3-small">
  The OpenAI embedding model for semantic search.
  
  **Used in**: `~/workspace/source/convex/llm.ts:8`
  
  **Supported values**:
  - `text-embedding-3-small` (default, 1536 dimensions, fast)
  - `text-embedding-3-large` (3072 dimensions, more accurate, slower)
  - `text-embedding-ada-002` (legacy, 1536 dimensions)
  
  **Example**: `text-embedding-3-large`
  
  <Warning>
  Changing the embedding model after data is already embedded will break similarity search. If you change this, you must re-run the embed phase for all persons.
  </Warning>
</ParamField>

## Next.js (Vercel) Variables

These variables are set in **Vercel's environment variables** (or `.env.local` for local development).

### Production/Preview Only

<ParamField path="CONVEX_DEPLOY_KEY" type="string" required="production">
  Production deploy key from Convex for CI/CD.
  
  **Where to get**: Convex Dashboard → Settings → Production Deploy Keys
  
  **Used in**: Vercel build process (see `~/workspace/source/.env.example:4`)
  
  **Purpose**: Allows Vercel to automatically push Convex functions during deployment
  
  **Environment**: Production and Preview only (not needed for local development)
  
  <Note>
  As noted in `~/workspace/source/.env.example:2-3`, this variable is NOT needed in local development. During `npm run dev`, Convex uses the dev deployment URL from `.env.local` (auto-generated by `convex dev`).
  </Note>
</ParamField>

### Auto-Generated (No Action Required)

<ParamField path="CONVEX_URL" type="string" auto>
  The Convex deployment URL (e.g., `https://your-project.convex.cloud`).
  
  **Auto-generated by**: 
  - `convex dev` (local, written to `.env.local`)
  - `convex deploy` (production, used by Vercel build)
  
  **Used in**: `~/workspace/source/components/ConvexClientProvider.tsx` (imported from environment)
  
  **You should NOT manually set this**. It's automatically managed by Convex CLI.
</ParamField>

## Configuration Files

### Next.js Configuration

**File**: `~/workspace/source/next.config.ts`

The Next.js config is minimal by default:

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  /* config options here */
};

export default nextConfig;
```

You can customize this to add:

- **Image domains**: If you're loading external images, add to `images.remotePatterns`
- **Redirects**: Custom URL redirects
- **Headers**: Custom HTTP headers (CORS, CSP, etc.)
- **Environment variables**: Expose variables to the browser (prefixed with `NEXT_PUBLIC_`)

**Example**: Allow loading images from any domain (useful for timeline photos):

```typescript
const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '**',
      },
    ],
  },
};
```

### Package Scripts

**File**: `~/workspace/source/package.json:5-12`

Available npm scripts:

<CodeGroup>
```json package.json
{
  "scripts": {
    "dev": "npm-run-all --parallel dev:frontend dev:backend",
    "dev:frontend": "next dev",
    "dev:backend": "convex dev",
    "predev": "convex dev --until-success && convex dashboard",
    "build": "next build",
    "start": "next start",
    "lint": "eslint . --ignore-pattern \"convex/_generated/**\""
  }
}
```
</CodeGroup>

**Key scripts**:

- `npm run dev`: Starts both Next.js dev server (port 3000) and Convex dev mode in parallel
- `npm run predev`: Ensures Convex is ready and opens the dashboard before starting dev mode
- `npm run build`: Production build (used by Vercel)
- `npm run lint`: Run ESLint (configured to ignore Convex generated files)

## Pipeline Configuration

The ingestion pipeline behavior can be customized by editing `~/workspace/source/convex/pipeline.ts`.

### Discovery Phase Parameters

**Location**: `~/workspace/source/convex/pipeline.ts:751-791`

<CodeGroup>
```typescript pipeline.ts
// Number of search queries to generate
const searchQueries = (queryPlan.queries ?? []).slice(0, 14);

// Fallback queries if LLM fails
if (searchQueries.length === 0) {
  searchQueries.push(
    { query: `"${person.name}" biography profile`, type: "web" },
    { query: `"${person.name}" interview`, type: "web" },
    { query: `"${person.name}" interview talk`, type: "video" },
    { query: `"${person.name}" rare early`, type: "video" },
  );
}
```
</CodeGroup>

**Customizable**:
- `slice(0, 14)`: Max number of search queries (default: 14)
- Fallback query patterns

### Search Result Limits

**Location**: `~/workspace/source/convex/pipeline.ts:252-259`

<CodeGroup>
```typescript pipeline.ts
const results = await exaSearch(q.query, {
  numResults: isVideo ? 5 : 8,  // Videos: 5, Web: 8
  text: !isVideo,
  includeDomains: isVideo ? ["youtube.com", "youtu.be"] : undefined,
```

</CodeGroup>

**Customizable**:
- `numResults`: Number of results per query (video: 5, web: 8)
- `includeDomains`: Restrict to specific domains

### Staging Parameters

**Location**: `~/workspace/source/convex/pipeline.ts:953`

<CodeGroup>
```typescript pipeline.ts
const stageJson = await generateJson(
  "You create biographical life stages by age-era first, then refine with context shifts. Output strict JSON object with key 'stages'. Each stage: order, title, ageStart, ageEnd, dateStart, dateEnd, eraSummary, worldviewSummary, turningPoints (array), confidence. Title MUST look like '[18-24] - Startup Operator: Early Builder'. ### Embedding Parameters
```

**Location**: `~/workspace/source/convex/pipeline.ts:127-138, 1251-1252`

<CodeGroup>
```typescript pipeline.ts
function splitChunks(text: string, maxChars = 1200): string[] {
  const clean = sanitizeText(text).replace(/\s+/g, " ").trim();
  if (!clean) return [];
  const out: string[] = [];
  let cursor = 0;
  while (cursor < clean.length) {
    const end = Math.min(clean.length, cursor + maxChars);
    out.push(clean.slice(cursor, end));
    cursor = end;
  }
  return out;
}

const CHUNKS_PER_SOURCE = 10;
const BATCH_SIZE = 20;
```
</CodeGroup>

**Customizable**:
- `maxChars`: Chunk size for embeddings (default: 1200 chars)
- `CHUNKS_PER_SOURCE`: Max chunks per source (default: 10)
- `BATCH_SIZE`: Batch size for embedding API calls (default: 20)

<Warning>
Increasing `CHUNKS_PER_SOURCE` will increase embedding costs and database size, but improve semantic search quality.
</Warning>

## Local Development Setup

### Initial Setup

<Steps>
  <Step title="Install Dependencies">
    ```bash
    cd ~/workspace/source
    npm install
    ```
  </Step>
  
  <Step title="Login to Convex">
    ```bash
    npx convex login
    ```
  </Step>
  
  <Step title="Start Development Mode">
    ```bash
    npm run dev
    ```
    
    This:
    - Starts Convex dev deployment
    - Generates `.env.local` with `CONVEX_URL`
    - Starts Next.js on `http://localhost:3000`
  </Step>
  
  <Step title="Set Convex Environment Variables">
    Go to [dashboard.convex.dev](https://dashboard.convex.dev), select your dev deployment, and add:
    - `OPENAI_API_KEY`
    - `EXA_API_KEY`
  </Step>
</Steps>

### Local Environment File

After running `convex dev`, you'll have a `.env.local` file:

```bash
# Auto-generated by Convex
CONVEX_URL=https://your-dev-deployment.convex.cloud
NEXT_PUBLIC_CONVEX_URL=https://your-dev-deployment.convex.cloud
```

<Note>
Do NOT commit `.env.local` to git. It's already in `.gitignore`.
</Note>

## Monitoring and Logs

### Convex Dashboard

Access real-time logs and database state:

1. Go to [dashboard.convex.dev](https://dashboard.convex.dev)
2. Select your project
3. View:
   - **Logs**: Real-time function execution logs
   - **Data**: Browse database tables (`persons`, `sources`, `stages`, `chunks`)
   - **Functions**: See all deployed functions and their schemas

### Vercel Logs

Access deployment and runtime logs:

1. Go to your project in Vercel dashboard
2. Click on a deployment
3. View:
   - **Build Logs**: npm install, Convex deploy, Next.js build
   - **Function Logs**: Next.js API route logs (if any)
