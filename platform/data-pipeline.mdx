## Pipeline Overview

The LORE Timeline ingestion pipeline transforms a person's name into a rich, searchable biography through 5 sequential phases. Each phase completes, persists results, and schedules the next phase.

```
User Input: "Name"
      |
      v
┌─────────────────────────────────────────────┐
│  PHASE 1: DISCOVER                          │
│  • Generate search queries with LLM         │
│  • Execute searches via Exa AI              │
│  • Vet sources for relevance                │
│  Output: 10-30 source URLs                  │
└──────────────┬──────────────────────────────┘
               v
┌─────────────────────────────────────────────┐
│  PHASE 2: EXTRACT                           │
│  • Fetch full content from URLs             │
│  • Extract YouTube transcripts              │
│  • Strip HTML, sanitize text                │
│  Output: Raw text + transcripts             │
└──────────────┬──────────────────────────────┘
               v
┌─────────────────────────────────────────────┐
│  PHASE 3: STAGE                             │
│  • Generate life stages with LLM            │
│  • Map sources to stages                    │
│  • Deep research per stage                  │
│  Output: 3-7 stages + stage-source links    │
└──────────────┬──────────────────────────────┘
               v
┌─────────────────────────────────────────────┐
│  PHASE 4: EMBED                             │
│  • Split text into 1200-char chunks         │
│  • Generate embeddings (batch 20)           │
│  • Store vectors + citations                │
│  Output: 100-500 embedded chunks            │
└──────────────┬──────────────────────────────┘
               v
┌─────────────────────────────────────────────┐
│  PHASE 5: PUBLISH                           │
│  • Search for person images per stage       │
│  • Generate timeline cards                  │
│  • Mark person as "ready"                   │
│  Output: Timeline ready for display         │
└─────────────────────────────────────────────┘
```

<Note>
Each phase is **resumable** - if a timeout or error occurs, the next phase can be triggered manually without losing prior work.
</Note>

### Step 2: Execute Search Plan

Queries are executed via Exa AI's search API:

```typescript convex/pipeline.ts:214-274
async function executeSearchPlan(name: string, queries) {
  const seen = new Set<string>();
  const all: CandidateSource[] = [];
  
  for (const q of queries) {
    const isVideo = q.type === "video";
    const results = await exaSearch(q.query, {
      numResults: isVideo ? 5 : 8,
      text: !isVideo, // Get text snippets for non-video
      includeDomains: isVideo ? ["youtube.com", "youtu.be"] : undefined,
      startPublishedDate: q.dateStart,
      endPublishedDate: q.dateEnd,
    });
    
    // Deduplicate by URL
    for (const r of results) {
      if (seen.has(r.url)) continue;
      seen.add(r.url);
      all.push({
        url: r.url,
        title: r.title ?? r.url,
        type: classifySourceType(r.url),
        publishedAt: r.publishedDate,
        snippet: r.text?.slice(0, 400),
      });
    }
  }
  
  return all;
}
```

### Step 3: Vet Sources

The LLM validates each source is actually about the target person (not someone else with the same name):

```typescript convex/pipeline.ts:281-323
async function vetSources(name: string, sources, generateJson) {
  const BATCH = 15;
  const vetted: CandidateSource[] = [];
  
  for (let i = 0; i < sources.length; i += BATCH) {
    const batch = sources.slice(i, i + BATCH);
    const items = batch.map((s, idx) => ({
      idx,
      title: s.title,
      url: s.url,
      snippet: s.snippet?.slice(0, 200),
    }));
    
    const resultJson = await generateJson(
      `Given a target person's name and web sources, determine which sources are genuinely ABOUT that specific person.
      
      Output JSON: { "valid": [array of idx numbers] }
      
      Be strict: exclude different people with similar names, tangential mentions, or uncertain matches.`,
      JSON.stringify({ targetPerson: name, sources: items })
    );
    
    const parsed = parseJson(resultJson, {});
    const validSet = new Set(parsed.valid ?? []);
    
    for (let j = 0; j < batch.length; j++) {
      if (validSet.has(j)) vetted.push(batch[j]);
    }
  }
  
  return vetted;
}
```

<Warning>
Vetting is critical to avoid false positives. Common names like "Michael Jordan" require strict filtering to ensure we get the basketball player, not other people.
</Warning>

    ```typescript convex/pipeline.ts:558-632
    async function fetchYoutubeTranscript(videoId: string) {
      // Step 1: Fetch video page to find caption tracks
      const pageResponse = await fetch(
        `https://www.youtube.com/watch?v=${videoId}`,
        { headers: { "User-Agent": "Mozilla/5.0..." } }
      );
      
      const pageHtml = await pageResponse.text();
      const captionMatch = pageHtml.match(/"captionTracks":\s*(\[.*?\])/);
      
      if (!captionMatch) return "";
      
      const captionTracks = JSON.parse(captionMatch[1]);
      const track = captionTracks.find(t => 
        t.languageCode === "en"
      ) ?? captionTracks[0];
      
      // Step 2: Fetch transcript XML
      const captionUrl = track.baseUrl + "&fmt=srv3";
      const captionResponse = await fetch(captionUrl);
      const captionXml = await captionResponse.text();
      
      // Step 3: Parse text segments
      const segments: string[] = [];
      const regex = /<p[^>]*>(.*?)<\/p>/gi;
      let match;
      
      while ((match = regex.exec(captionXml)) !== null) {
        const text = match[1]
          .replace(/<[^>]+>/g, "")
          .replace(/&amp;/g, "&")
          .trim();
        if (text) segments.push(text);
      }
      
      return segments.join(" ").slice(0, 30000);
    }
    ```

### Image Extraction

Extract relevant images from HTML (portraits, photos) while filtering out logos, icons, and ads:

```typescript convex/pipeline.ts:420-488
function extractImageUrls(html: string, baseUrl: string, personName?: string) {
  const imgRegex = /<img[^>]*>/gi;
  const urls: string[] = [];
  const nameParts = (personName ?? "").toLowerCase().split(/\s+/);
  
  let match;
  while ((match = imgRegex.exec(html)) !== null) {
    const fullTag = match[0];
    const srcMatch = fullTag.match(/src=["']([^"']+)["']/i);
    if (!srcMatch?.[1]) continue;
    const src = srcMatch[1];
    
    // Skip data URIs, tracking pixels, icons, logos
    if (src.startsWith("data:") || 
        src.includes("1x1") || 
        src.includes("logo") ||
        src.endsWith(".svg")) continue;
    
    // Check size hints - skip small images
    const widthMatch = fullTag.match(/width=["']?(\d+)/i);
    if (widthMatch && parseInt(widthMatch[1]) < 150) continue;
    
    // Check alt text for person's name
    const altMatch = fullTag.match(/alt=["']([^"']*)["']/i);
    const altText = (altMatch?.[1] ?? "").toLowerCase();
    
    const nameInAlt = nameParts.some(part => altText.includes(part));
    const nameInSrc = nameParts.some(part => src.toLowerCase().includes(part));
    
    if (nameInAlt || nameInSrc || 
        src.includes("photo") || 
        altText.includes("portrait")) {
      urls.push(resolveUrl(src, baseUrl));
    }
  }
  
  return [...new Set(urls)].slice(0, 4);
}
```

## Phase 4: Embed

**Goal:** Split all source text into chunks, generate embeddings, and store for semantic search.

### Text Chunking

```typescript convex/pipeline.ts:127-138
function splitChunks(text: string, maxChars = 1200): string[] {
  const clean = sanitizeText(text).replace(/\s+/g, " ").trim();
  if (!clean) return [];
  
  const out: string[] = [];
  let cursor = 0;
  
  while (cursor < clean.length) {
    const end = Math.min(clean.length, cursor + maxChars);
    out.push(clean.slice(cursor, end));
    cursor = end;
  }
  
  return out;
}
```

<Tabs>
  <Tab title="Sanitization">
    ```typescript convex/pipeline.ts:118-125
    function sanitizeText(raw: string): string {
      return raw
        // Remove incomplete unicode escapes
        .replace(/\\u[0-9a-fA-F]{0,3}(?![0-9a-fA-F])/g, "")
        // Remove null bytes and control characters
        .replace(/[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]/g, "");
    }
    ```
  </Tab>
  
  <Tab title="Why 1200 chars?">
    **Rationale:**
    - Small enough for focused context
    - Large enough to capture full thoughts
    - Fits comfortably in embedding model context
    - ~200-250 tokens on average
    
    **Trade-offs:**
    - Smaller chunks (500) = more embeddings, higher cost, less context
    - Larger chunks (2000) = fewer embeddings, but may dilute semantic signal
  </Tab>
</Tabs>

### Batch Embedding

```typescript convex/pipeline.ts:1257-1318
const CHUNKS_PER_SOURCE = 10; // Limit to control costs
const BATCH_SIZE = 20; // Batch API calls

for (const source of sources) {
  // Skip if already embedded (resumable)
  if (embeddedSourceIds.has(String(source._id))) continue;
  
  const text = sanitizeText(source.rawText ?? source.transcriptText ?? "");

      { texts: batchTexts }

    for (let j = 0; j < batchTexts.length; j++) {
      await ctx.runMutation(internal.pipeline.insertChunk, {
        personId,
        sourceId: source._id,
        stageId: stageLinkMap.get(String(source._id)),
        text: batchTexts[j],
        embedding: embeddings[j] ?? [],
        citation: citationJson,
      });
    }
  }
}
```

### Embedding API Call

```typescript convex/llm.ts:77-107
export const embedTextBatch = action({
  args: { texts: v.array(v.string()) },
  handler: async (_ctx, args) => {
    if (args.texts.length === 0) return [];
    
    const response = await fetch("https://api.openai.com/v1/embeddings", {
      method: "POST",
      headers: {
        Authorization: `Bearer ${OPENAI_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        model: "text-embedding-3-small", // 1536 dimensions
        input: args.texts,
      }),
    });
    
    const data = await response.json();
    const sorted = data.data.sort((a, b) => a.index - b.index);
    return sorted.map(d => d.embedding); // number[][]
  },
});
```

<Warning>
**Cost Control:**
- Limit to 10 chunks per source (max 12,000 chars)
- Batch 20 texts per API call (reduces requests)
- `text-embedding-3-small` is 5x cheaper than `text-embedding-3-large`

For a person with 30 sources:
- 30 sources × 10 chunks = 300 chunks
- 300 chunks / 20 per batch = 15 API calls
- ~36,000 tokens = ~$0.001 (very cheap)
</Warning>

```typescript convex/pipeline.ts:495-535
async function searchPersonImages(
  name: string,
  stageTitle?: string
): Promise<string[]> {
  const query = stageTitle
    ? `${name} photo ${stageTitle.replace(/^\[.*?\]\s*-\s*/, "")}`
    : `${name} photo portrait`;
  
  const response = await fetch("https://api.exa.ai/search", {
    method: "POST",
    headers: { "x-api-key": EXA_API_KEY },
    body: JSON.stringify({
      query,
      numResults: 5,
      type: "auto",
    }),
  });
  
  const data = await response.json();
  const images: string[] = [];
  
  for (const result of data.results ?? []) {
    if (result.image?.startsWith("http")) {
      images.push(result.image);
    }
  }
  
  return [...new Set(images)].slice(0, 3);
}
```

```typescript convex/pipeline.ts:1379-1391
  await ctx.runMutation(internal.pipeline.markLatestRunningJobFailed, {
    personId,
    message: error instanceof Error ? error.message : "Unknown failure",
  });
  
  throw error;
}
```

### Manual Resume

If a phase times out or fails, manually resume from any phase:

```typescript
// From frontend or admin panel
await reingestPerson({
  personId,
  fromPhase: "embed", // Resume from embed phase
});
```

<Accordion title="Resume Logic Per Phase">
  - **Discover:** Replaces sources (destructive)
  - **Extract:** Skips sources with `rawText` or `transcriptText`
  - **Stage:** Skips if stages already exist
  - **Embed:** Skips sources already in `chunks` table
  - **Publish:** Idempotent (can re-run safely)
</Accordion>

## Performance & Timing

| Phase | Typical Duration | Bottleneck |
|-------|-----------------|------------|
| Discover | 30-60 seconds | LLM query generation + Exa searches |
| Extract | 45-90 seconds | Fetching URLs, YouTube transcripts |
| Stage | 60-120 seconds | LLM stage generation + deep research |
| Embed | 30-90 seconds | OpenAI embedding API (batched) |
| Publish | 15-30 seconds | Exa image search per stage |
| **Total** | **3-8 minutes** | Network I/O + LLM latency |

<Note>
Timing varies based on:
- Number of sources discovered (10-50)
- Number of stages generated (3-7)
- YouTube transcript availability
- API response times
</Note>
