LORE Timeline's AI chat feature lets you have conversations with a person as if you're speaking to them during a specific era of their life. Every response is grounded in real source evidence using Retrieval-Augmented Generation (RAG).

## How It Works

<Steps>
  <Step title="Select an Era">
    Click the chat button on any life stage to start a conversation grounded in that period
  </Step>
  
  <Step title="Ask a Question">
    Your question is converted into a vector embedding for semantic search
  </Step>
  
  <Step title="Semantic Retrieval">
    The system searches through indexed text chunks using cosine similarity to find relevant evidence
  </Step>
  
  <Step title="Context Assembly">
    Top-ranked chunks are formatted with citations and combined into a context window
  </Step>
  
  <Step title="Voice Matching">
    Real quotes from the person are extracted to match their speaking style
  </Step>
  
  <Step title="Generate Response">
    The LLM responds in first person, mimicking the person's voice with inline citations
  </Step>
</Steps>

## Semantic Search with RAG

LORE uses vector embeddings to find relevant evidence for each question:

```typescript
export const scoreAndSelectChunks = action({
  args: {
    personId: v.id("persons"),
    stageId: v.id("stages"),
    query: v.string(),
    topK: v.optional(v.number()),
  },
  handler: async (ctx, args) => {
    const topK = args.topK ?? 8;
    
    // Convert question to vector embedding
    const qEmbedding = await ctx.runAction(api.llm.embedText, { 
      text: args.query 
    });

    // Get all chunks for this life stage
    const stageChunks = await ctx.runQuery(
      internal.pipeline.listChunksByStage,
      { stageId: args.stageId },
    );
    
    // Score each chunk by cosine similarity
    const scoredStage = stageChunks
      .map((chunk) => ({ 
        chunk, 
        score: cosineSimilarity(qEmbedding, chunk.embedding) 
      }))
      .sort((a, b) => b.score - a.score)
      .slice(0, topK);

    let usedFallback = false;
    let result = scoredStage;

    // Fallback: if stage evidence is sparse, search all eras
    if (scoredStage.length < Math.max(3, Math.floor(topK / 2))) {
      usedFallback = true;
      const allChunks = await ctx.runQuery(
        internal.pipeline.listChunksByPerson,
        { personId: args.personId },
      );
      result = allChunks
        .map((chunk) => ({ chunk, score: cosineSimilarity(qEmbedding, chunk.embedding) }))
        .sort((a, b) => b.score - a.score)
        .slice(0, topK);
    }

    return {
      usedFallback,
      chunks: result.map((item) => ({
        id: item.chunk._id,
        text: item.chunk.text,
        citation: item.chunk.citation,
        score: item.score,
      })),
    };
  },
});
```

<Info>
  **Stage-Scoped Retrieval**: By default, the system only searches chunks from the selected life stage. This ensures responses reflect that specific era's worldview.
</Info>

## Cosine Similarity Scoring

The system ranks text chunks by semantic relevance:

```typescript
function cosineSimilarity(a: number[], b: number[]): number {
  if (a.length === 0 || b.length === 0 || a.length !== b.length) return 0;
  
  let dot = 0;
  let magA = 0;
  let magB = 0;
  
  for (let i = 0; i < a.length; i += 1) {
    dot += a[i] * b[i];
    magA += a[i] * a[i];
    magB += b[i] * b[i];
  }
  
  if (!magA || !magB) return 0;
  return dot / (Math.sqrt(magA) * Math.sqrt(magB));
}
```

Scores range from 0 to 1, where:
- **0.9-1.0**: Nearly identical meaning
- **0.7-0.9**: Highly relevant
- **0.5-0.7**: Somewhat relevant
- **< 0.5**: Low relevance

## Voice Matching

The system extracts real quotes from source evidence to match the person's speaking style:

```typescript
const voiceSamples: string[] = [];

for (const chunk of retrieval.chunks.slice(0, 5)) {
  const text = chunk.text;
  
  // Extract quoted speech (likely the person talking)
  const quoteMatches = text.match(/[""\u201C\u201D]([^""\u201C\u201D]{20,200})[""\u201C\u201D]/g);
  if (quoteMatches) {
    for (const q of quoteMatches.slice(0, 2)) {
      voiceSamples.push(q.replace(/[""\u201C\u201D]/g, ""));
    }
  }
  
  // Extract first-person speech from transcripts
  const sentences = text.split(/[.!?]+/).filter((s) => {
    const trimmed = s.trim().toLowerCase();
    return (
      (trimmed.startsWith("i ") || 
       trimmed.startsWith("we ") || 
       trimmed.startsWith("my ")) &&
      trimmed.length > 30 && 
      trimmed.length < 250
    );
  });
  
  for (const s of sentences.slice(0, 2)) {
    voiceSamples.push(s.trim());
  }
}

const uniqueVoice = [...new Set(voiceSamples)].slice(0, 6);
```

## Cross-Stage Fallback

When a stage has limited evidence, the system expands retrieval:

<Accordion title="Fallback Logic">
  ```typescript
  // If stage has fewer than half the requested chunks, use cross-stage fallback
  if (scoredStage.length < Math.max(3, Math.floor(topK / 2))) {
    usedFallback = true;
    
    // Search ALL chunks across all life stages
    const allChunks = await ctx.runQuery(
      internal.pipeline.listChunksByPerson,
      { personId: args.personId },
    );
    
    result = allChunks
      .map((chunk) => ({ chunk, score: cosineSimilarity(qEmbedding, chunk.embedding) }))
      .sort((a, b) => b.score - a.score)
      .slice(0, topK);
  }
  ```
  
  When fallback is used, the user sees a banner:
  
  > "Cross-stage retrieval was used — evidence for this specific era was limited."
</Accordion>

## Chat Session Management

Conversations are scoped to both person+stage AND client:

```typescript
const session = await ctx.db
  .query("chatSessions")
  .withIndex("by_person_stage_client", (q) =>
    q.eq("personId", personId)
     .eq("stageId", stageId)
     .eq("clientId", clientId),
  )
  .first();

if (!session) {
  sessionId = await ctx.runMutation(api.persons.createChatSession, {
    personId,
    stageId,
    clientId,
  });
}
```

<Info>
  **Isolated Sessions**: Each life stage has its own chat history. Switching stages starts a new conversation grounded in that era's evidence.
